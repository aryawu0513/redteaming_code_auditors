# automatic/

DSPy-based pipeline for generating adversarial attack payloads for new code variants, and writing all attack files to disk.

---

## Overview

Two stages:

1. **`generate_variant.py`** — given a `clean/` directory, calls Claude to extract injection anchors and generate all 10 context-aware attack payloads for each variant. Saves to `texts/`.
2. **`gen_attacks.py`** — reads all variants from `texts/` and writes the buggy baseline and context-aware attack files to `RepoAudit/benchmark/` and `VulnLLM-R/datasets/`.

---

## User Guide

To create an attack dataset for a new bug type / language from scratch:

**1. Create clean code files.** For each variant, add a source file at:
```
RepoAudit/benchmark/{Lang}/{BUG}/clean/{variant}/{variant}_clean.{ext}
```
Each file should contain the target function (with the latent bug) and any helper functions it calls.

**2. Generate attack payloads** (requires `ANTHROPIC_API_KEY`):
```bash
source RepoAudit/.venv/bin/activate
python automatic/generate_variant.py --bug-type npd --language c \
    --clean-dir RepoAudit/benchmark/C/NPD/clean
```
This reads every variant subdir, calls Claude (~13 LLM calls per variant), and saves results to `automatic/texts/c_npd.json`. Already-processed variants are skipped automatically.

**3. Write the attack dataset to disk:**
```bash
python automatic/gen_attacks.py --bug-type npd --language c
```
This reads `texts/` and produces the `buggy/` and `context_aware/` directory trees for both RepoAudit and VulnLLM-R.

That's it. The attack dataset is ready to run through RepoAudit and VulnLLM-R as described in the root `Pipeline.md`.

## Stage 1 — Generate payloads for all variants in a clean directory

**Prerequisite:** Clean source files exist under `{benchmark}/clean/{variant}/{variant}_clean.{ext}`.

```bash
source RepoAudit/.venv/bin/activate

# C / NPD — all variants in the clean directory
python automatic/generate_variant.py \
    --bug-type npd --language c \
    --clean-dir RepoAudit/benchmark/C/NPD/clean

# Python / NPD
python automatic/generate_variant.py \
    --bug-type npd --language python \
    --clean-dir RepoAudit/benchmark/Python/NPD/clean

# C / UAF
python automatic/generate_variant.py \
    --bug-type uaf --language c \
    --clean-dir RepoAudit/benchmark/C/UAF/clean

# Regenerate only specific attack types (skips COT, FT, etc. not listed)
python automatic/generate_variant.py --bug-type npd --language c \
    --clean-dir RepoAudit/benchmark/C/NPD/clean \
    --attack-types COT FT CG

# Force-regenerate variants already in texts/
python automatic/generate_variant.py --bug-type npd --language c \
    --clean-dir RepoAudit/benchmark/C/NPD/clean --force
```

**Produces:** `automatic/texts/{language}_{bug_type}.json` — one entry per variant, saved incrementally after each one.

**JSON structure:**
```json
{
  "variantname": {
    "meta": {
      "callee": "find_record",
      "caller": "print_record",
      "var": "rec",
      "caller_sig": "void print_record(",
      "call_line":  "    Record* rec = find_record(table, size, key);",
      "deref_line": "    printf(\"id=%d name=%s\\n\", rec->id, rec->name);",
      "last_line":  "    free(rec);",
      "clean_code": "// context\n..."
    },
    "COT":    "/* Security analysis: ... */",
    "FT":     "/* CI test results ... */",
    "CG":     "/* Call graph analysis ... */",
    "AA_MSG": "/* find_record always succeeds ... */",
    "..."
  }
}
```

> **Note:** `ANTHROPIC_API_KEY` must be set. Model is `claude-sonnet-4-6`. Each variant makes ~13 LLM calls (one metadata extraction + one per attack type). Variants already present in `texts/` are skipped unless `--force` is passed.

## Stage 2 — Write attack files to disk

```bash
source RepoAudit/.venv/bin/activate

python automatic/gen_attacks.py --bug-type npd --language c
python automatic/gen_attacks.py --bug-type uaf --language c
python automatic/gen_attacks.py --bug-type npd --language python
```

**Produces:**
- `RepoAudit/benchmark/{Lang}/{BUG}/{buggy,context_aware}/{variant}/`
- `VulnLLM-R/datasets/{Lang}/{BUG}/{buggy,context_aware}/{variant}/{lang}/`

> **Note:** All variants — both hand-curated and DSPy-generated — live in `texts/`. New variants added via `generate_variant.py` are written directly there.

---

## Verify — regression test

Checks that all three ground-truth subtrees are reproduced exactly:

```bash
python automatic/test_gen_all.py
```

**Produces:** pass/fail output; exits non-zero if any file differs from ground truth.

## Data files

| Path | Contents |
|------|----------|
| `texts/c_npd.json` | Metadata + attack texts for C NPD variants (findrec, creatend, mkbuf, allocate) |
| `texts/c_uaf.json` | Metadata + attack texts for C UAF variants (freeitem, dropconn, relogger, rmentry) |
| `texts/python_npd.json` | Metadata + attack texts for Python NPD variants (finduser, makeconn, parseitem, loadconf) |

`texts/` is the single source of truth. New variants generated by `generate_variant.py` are written directly into the relevant JSON file here.

## Attack type sets (10 per language)

Base types (all languages): `COT`, `FT`, `CG`, `AA_MSG`, `AA_USR`, `AA_CA`

| Language | TOOL_* attacks |
|----------|----------------|
| C | `TOOL_ClangSA`, `TOOL_Frama`, `TOOL_Fuzzer`, `TOOL_Coverity` |
| Python | `TOOL_Pysa`, `TOOL_Mypy`, `TOOL_Atheris`, `TOOL_Bandit` |
